# 🔗 LinkRoom — B2B Relationship Intelligence Hub

> An AI-powered collaboration platform for managing B2B partnerships through a real-time transaction ledger, semantic search, and automated ecosystem governance.

---

## 📌 Overview

**LinkRoom** is a full-stack web application designed to help organizations **track, manage, and analyze their B2B relationships** in a structured and intelligent way. It acts as a shared workspace ("room") between two organizations — where every communication, meeting, document exchange, and email is logged as a **transaction event** on a chronological ledger.

At the end of every event, an **AI Governance Engine** (powered by Google Gemini) analyzes the full relationship history and produces a **Global Executive Briefing** — a concise summary of collaboration health, stagnation risks, and recommended action items.

## 🖥️ Screenshots

> **Home — Relationship Graph**

The landing page displays an interactive node-graph where each organization is a node, and each active topic (project/deal) is a glowing curved connection line between them. Hovering over a connection reveals the project topic.

> **Room — Transaction Ledger**

A chat-style event ledger that shows the newest events at the bottom. Users can scroll up to access history, upload documents, schedule meetings, and send emails — all persisted in Firestore.

> **Global Executive Briefing (AI Panel)**

A real-time AI summary panel on the right side of the ledger screen, powered by Gemini 2.5 Flash, showing relationship health, stagnation risk score, and pending action items.

## ✨ Key Features

| Feature | Description |
|---|---|
| **Relationship Graph** | Interactive node graph visualizing all B2B relationships and their active project streams |
| **Transaction Ledger** | A chat-style chronological feed of all collaboration events (meetings, emails, paperwork) |
| **AI Executive Briefing** | Automated relationship health summaries generated by Google Gemini 2.5 Flash |
| **Document Storage** | Upload and persist documents/images directly in Firestore (Base64 encoded) with one-click download |
| **Meeting Scheduling** | Log Google Meet or Microsoft Teams links directly to the ledger |
| **Email Tracking** | Record email threads (sender, recipient, subject, body) as ledger events |
| **Semantic Search** | AI-powered vector search using Gemini Embedding 2 to find relevant events from natural language queries |
| **Search-to-Event Navigation** | Click a search result to automatically scroll and highlight the exact event in the ledger |
| **Dual Room System** | Each relationship has a **Public Shared Room** (visible to both parties) and an **Internal Private Room** |
| **Smart AI Polling** | The UI automatically detects when AI analysis is complete and updates without requiring a manual refresh |

## 🚀 Getting Started

### Prerequisites
- Node.js 20+
- A Google Cloud Project with Firestore enabled
- A Google AI Studio API Key (for Gemini)
- A Google Cloud Service Account key (JSON)

### 1. Clone the repository
```bash
git clone https://github.com/JokerHin/RoomLink.git
cd RoomLink
```

### 2. Install dependencies
```bash
# Frontend
npm install

# GCP Worker
cd gcp-worker
npm install
cd ..
```

### 3. Configure environment variables

Create a `.env.local` file in the root directory:

```env
NEXT_PUBLIC_GCP_PROJECT_ID="your-gcp-project-id"
GOOGLE_APPLICATION_CREDENTIALS="/absolute/path/to/your-service-account-key.json"
GEMINI_API_KEY="your-google-ai-studio-api-key"
```

### 4. Run the development server
```bash
# Terminal 1: Frontend
npm run dev

# Terminal 2: GCP Worker
cd gcp-worker
npm run build
npm run start

# Terminal 3: Local PubSub Bridge
cd gcp-worker
npm run bridge
```

### 5. Open the app
Navigate to [http://localhost:3000](http://localhost:3000)

## 🤖 AI Pipeline

Every time an event is added to the ledger, the following pipeline runs:

1. **Embedding Generation** — The event text is embedded using `gemini-embedding-2` for semantic search indexing.
2. **Chunk Persistence** — The event (including any file data in Base64) is saved to Firestore's `ledger_chunks` collection.
3. **Ecosystem Analysis** — The full relationship history is sent to `gemini-2.5-flash` with a structured prompt requesting a JSON governance report.
4. **Summary Persistence** — The AI output (`summary_markdown`, `relationship_health_score`, `stagnation_risk`, `action_items`) is saved to `room_summaries` and displayed in the UI.

The AI model cascade tries `gemini-2.0-flash-lite` → `gemini-2.0-flash` → `gemini-2.5-flash` → `gemini-2.5-pro` to ensure maximum uptime across different API quota tiers.

## 🔐 Security Notes

- **Never commit** `linkroom-key.json` or any `*-key.json` files. They are excluded via `.gitignore`.
